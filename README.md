# Abdullah-# نظام إدارة محل (Offline Smart Inventory)

## نظرة عامة
هذا مشروع تطبيق ويب يعمل بدون إنترنت، مخصص لإدارة محلات البيع بالتجزئة الصغيرة والمتوسطة.
يهدف إلى تسجيل حركة البضائع اليومية، متابعة المخزون، وإصدار تقارير يومية تلقائية لصاحب المحل.

النظام ليس نظام كاشير، بل نظام متابعة وتحليل داخلي يعتمد على إدخال البيانات يدويًا أو عبر تصوير الفواتير.

---

## الهدف من المشروع
- تسجيل المنتجات داخل المحل
- متابعة كمية المخزون لكل صنف
- تسجيل المبيعات اليومية
- تسجيل المشتريات والفواتير
- عرض تقرير يومي تلقائي
- تنبيه الأصناف الناقصة
- تنظيم الأصناف ضمن بوابات (فئات)

---

## المميزات الحالية
- واجهة مستخدم حديثة ومتجاوبة
- دعم العمل بدون إنترنت
- عرض بوابات الأصناف (Categories)
- عرض المنتجات مع الصور والسعر والمخزون
- نظام تنبيهات للمخزون المنخفض
- لوحة تقرير يومي
- تصميم مناسب للجوال

---

## بنية البيانات (Database Schema)

### Categories
- الفئات الأساسية مثل: جوالات، شواحن، سماعات

### Products
- اسم المنتج
- الكود
- السعر
- كمية المخزون
- صورة المنتج
- تنبيه الحد الأدنى للمخزون

### Sales
- تسجيل المبيعات اليومية
- الكمية
- السعر وقت البيع
- التاريخ

### Purchases
- تسجيل المشتريات
- الفواتير
- الصور المرفقة

### Daily Reports
- إجمالي المبيعات
- إجمالي الأرباح
- عدد العمليات
- الأصناف الناقصة

### Uploaded Images
- تخزين صور الفواتير أو الملاحظات

---

## طريقة الاستخدام
1. فتح ملف `index.html` في المتصفح
2. تصفح البوابات (Categories)
3. عرض المنتجات وإدخال البيانات (لاحقًا عبر قاعدة بيانات)
4. مراجعة التقرير اليومي
5. متابعة المخزون والنواقص

---

## التطوير المستقبلي
- ربط قاعدة بيانات SQLite أو IndexedDB
- إضافة نظام إدخال منتجات ديناميكي
- دعم رفع الصور من الجهاز
- إنشاء تقارير أسبوعية وشهرية
- تحويل التطبيق إلى APK باستخدام Capacitor
- دعم OCR لقراءة الفواتير من الصور

---

## المتطلبات
- متصفح حديث (Chrome / Edge / Firefox)
- لا يحتاج إنترنت
- لا يحتاج سيرفر

---

## ملاحظات
المشروع مصمم ليكون بسيط وسريع الاستخدام وليس نظام محاسبي معقد.
مناسب لأصحاب المحلات الذين يريدون تتبع المخزون والمبيعات بشكل عملي وسريع
